Brand Identity System

< CodePath />

A coding bootcamp that transforms career-switchers into full-stack developers through project-based learning. Built for the ambitious 22-35 generation across India and Southeast Asia.

01 -- Archetype

Brand Personality

The Magician archetype drives transformation and possibility. The Explorer influence adds ambition, independence, and a pioneering edge. Together they create a brand that makes career change feel like an adventure rather than a risk.

The Magician + Explorer

Transformative Pioneering Energetic Visionary

Brands that embody this blend make the impossible feel within reach while encouraging bold leaps. They turn complexity into clarity and give their audience the tools to forge a new path. The emotional core is about unlocking potential that already exists -- the code was always inside you.

Voice Guidelines

// do

Use direct, action-oriented language. "Build your first API in 48 hours." Speak in commands and invitations, not descriptions.

// do

Reference the transformation journey. Acknowledge where learners are and where they are headed. "From Excel formulas to React components."

// don't

Avoid corporate jargon or empty hype. Never say "synergize," "leverage," or "disrupt." The brand sounds like a sharp senior dev, not a pitch deck.

// don't

Never sound condescending about career-switchers. No "even non-techies can..." framing. Everyone here is already capable -- they just need direction.

02 -- Logo & Wordmark

Wordmark System

The wordmark is a typographic mark rendered in Fira Code. The angle brackets and self-closing slash < /> mirror JSX and HTML tag syntax, making the brand name a piece of code itself. The brackets are always rendered in the warm accent (chartreuse) to distinguish the structural element from the name.

06 -- Don't

Negative Rules

These are explicit violations of the brand system. Each one undermines the technical credibility and energy that define the brand.

Don't
< CodePath />

Never stretch or distort the wordmark. The monospaced grid is precise by nature -- respect it.

Don't
< CodePath />

Never apply gradients to the wordmark. The logo must always use flat, solid colors for maximum clarity.

Don't
< CodePath />

Never use unapproved fonts for the wordmark. Arial, Helvetica, and system sans-serifs destroy the code-native identity.

Don't
This body text at 10px is not readable on any screen at any distance.

Never set body text below 14px (0.875rem). Accessibility and readability are non-negotiable.

Don't
< CodePath />

Never use grey as a primary color or apply it to the wordmark. Grey signals indifference -- the opposite of this brand.

Don't

Never remove focus indicators from interactive elements. Keyboard and assistive technology users depend on them.
